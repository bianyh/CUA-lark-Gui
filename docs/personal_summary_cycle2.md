# 第二周期（4.26-4.28）

## 一、核心产出

本周期最主要的成果，是把 CUA-Lark 的真实执行范围收敛到飞书窗口内。

之前截图和点击更偏通用桌面自动化，这一版改成先定位飞书窗口，即使飞书在 Windows 后台或最小化，也会先恢复并激活窗口，再只截取飞书窗口区域。模型返回的坐标被视为窗口内坐标，执行器会自动换算成屏幕绝对坐标后点击。

同时补充了更清晰的控制台日志，真实执行时可以看到用例、步骤、截图、动作、验证、恢复和报告生成过程，方便调试。

## 三、过程复盘与沉淀

### 1. 主要完成了什么？

主要完成了飞书窗口定位、窗口截图、后台窗口激活、坐标映射和控制台日志增强。实现时优先用 `pygetwindow` 查找并激活窗口，失败时再尝试 `pywinauto`，保证 Windows 环境下更稳。

### 2. 遇到的问题和解决方式

核心问题是 GUI 自动化不能继续假设“截图坐标就是屏幕坐标”。飞书窗口截图后，模型看到的是窗口内坐标，所以我在 `Observation` 里增加了 `screen_bounds`，再由执行器统一把窗口内坐标转换成屏幕坐标。

另一个小问题是 Rich 控制台会把 `[run]` 这类日志标签当作 markup 解析，导致日志显示不完整。后来改成禁用 markup 输出，日志就稳定了。

### 3. 可复用沉淀

这次沉淀下来的窗口作用域处理方式后续可以直接复用到其他桌面应用：先定位目标窗口，只截目标窗口，再把模型坐标从窗口坐标系映射回屏幕坐标系。

## 四、本周随手记

真实 GUI Agent 里，“看哪里”和“点哪里”必须用同一个坐标体系。只要截图范围变了，坐标映射就必须显式建模，不能靠经验猜。
